# Learning Rule: Topic Mastery Format

## File Header Rule:
- **ALWAYS start every file with a detailed overview block** that includes:
  - Day number and topic title
  - What you will learn today (list every concept)
  - Why it matters (real-world use)
  - Estimated time to complete

## Structure for Each Topic (5 Sections — Always in This Order):

### 1. Complete Explanation
- Define the topic clearly with simple language
- Explain WHY it's important and when to use it
- Break down key concepts step-by-step
- Include visual diagrams or analogies where helpful
- Add inline comments explaining each line of code
- **Explain EVERY concept, even the simplest ones** (e.g., what is print, what is f-string, what are curly braces, what is a dot, etc.)
- **Assume the user knows NOTHING** — never skip explaining something because it seems obvious
- **Use real-life analogies** to make abstract concepts easy to understand
- **Show what happens step-by-step** — break one line of code into multiple small steps if needed

### 2. Multiple Examples (Minimum 3)
- **Example 1**: Basic/Simple use case
- **Example 2**: Intermediate with real-world scenario
- **Example 3**: Advanced/Complex application
- Each example should build on previous knowledge
- Show different variations and edge cases
- **Every concept taught in section 1 MUST be clearly demonstrated in the examples**

### 3. Comprehensive Exercises (Easy + Medium + Hard + Bonus)
- **Easy (3-5 exercises)**: Test basic understanding
- **Medium (3-5 exercises)**: Apply concepts in new ways
- **Hard (2-3 exercises)**: Combine multiple concepts
- Include solutions with explanations
- **Bonus Challenges** go INSIDE this section, after Hard exercises

### 4. Practice Tips
- Common mistakes to avoid
- Best practices and conventions
- Related topics to explore next
- Quick reference summary at the end

### 5. Practice Exercises (Minimum 5)
- Provide at least 5 practice exercises WITHOUT solutions
- User should write code themselves to solve these
- Range from easy to medium difficulty
- Focus on hands-on coding practice
- These are separate from the solved exercises above
- **Every concept taught in section 1 MUST appear in at least one practice exercise**

## File Management Rules:

### Before Creating Files:
- **ALWAYS check if file already exists**
- **If file exists, ASK user permission before editing/replacing**
- Avoid creating duplicate files
- Use descriptive filenames to prevent confusion

### Before Moving to Next Topic:
- **ALWAYS ask user permission before creating next day's file**
- Confirm user has completed current topic
- Example: "Ready to move to Day X? Shall I create the next file?"
- Wait for explicit confirmation before proceeding
