## 🐍 Python Skills Tiers for ML Engineers

### 🔴 Must-Know (Non-Negotiable)
- Variables, data types (int, float, str, bool)
- Control flow: if/elif/else, for, while, break, continue
- Functions: parameters, return values, default args, *args, **kwargs
- Data structures: lists, dicts, sets, tuples — and when to use each
- List/dict/set comprehensions
- String manipulation & f-strings
- File I/O (read/write files)
- Error handling: try/except/finally
- Imports & modules
- **OOP:** Classes, `__init__`, instance vs class attributes, methods, self, inheritance, dunder methods
- **Libraries:** NumPy, Pandas, Matplotlib/Seaborn

### 🟡 Should-Know (Makes You Competitive)
- Lambda functions, map, filter, zip, enumerate
- Generators & iterators
- Decorators (at least understand them)
- Context managers (`with` statement)
- Type hints
- Virtual environments (venv, conda)
- Working with JSON, CSV, APIs (`requests` library)
- Basic regex
- `os` and `pathlib` for file/directory operations
- Logging basics
- **Git:** branches, merging, pull requests, resolving conflicts
- **SQL:** SELECT, JOIN, GROUP BY, window functions, CTEs

### 🟢 Nice-to-Know (Sets You Apart)
- Async basics (asyncio)
- Multiprocessing / threading
- Writing packages (setup.py / pyproject.toml)
- Unit testing (pytest)
- Profiling & optimization
- Dataclasses
- Abstract base classes

### 🔵 ML-Specific Python
- NumPy → tensor/array math, foundation of everything
- Pandas → data loading, cleaning, exploration
- Scikit-learn → ML models, pipelines, preprocessing
- PyTorch → deep learning (tensors, autograd, nn.Module, DataLoader)
- Hugging Face → pretrained models, tokenizers
- FastAPI/Flask → serving models as APIs
